# RDE 双粒度分歧交互（DDI）实验结果汇总

## 1. 摘要

本文档汇总 RDE + DDI 在 RSTPReid 上的两轮完整实验：

1. `qwen3-vl-flash-2025-10-15`
2. `qwen3.6-flash-2026-04-16`

两轮实验使用完全相同的 RDE checkpoint、200-query 子集、图库、随机种子、DDI 参数和提示词版本，因此结果可以直接比较。

主要结论如下：

- DDI 在两个 Qwen 模型上均能提升 RDE 的 Rank-1 和 mAP，说明双粒度分歧确实可以转化为有效的测试时补充信息。
- `qwen3-vl-flash` 的最佳结果出现在第 2 轮，表现较稳定；最佳 Rank-1 为 71.50%，mAP 为 55.11%。
- `qwen3.6-flash` 的最佳结果提前到第 1 轮；Rank-1 为 71.50%，Rank-10 为 91.00%，mAP 为 55.15%。
- `qwen3.6-flash` 更积极地确认属性，带来更大的早期收益，也造成更多查询恶化和多轮漂移。
- 当前最合适的部署配置是：`qwen3.6-flash-2026-04-16`，仅执行 1 轮 DDI。

## 2. 实验设置

| 项目 | 设置 |
|---|---|
| 数据集 | RSTPReid |
| 快速实验查询 | 200 个测试身份，每个身份固定抽取 1 条图文配对 |
| 图库 | 完整测试图库，1,000 张图像 |
| 随机种子 | 42 |
| RDE checkpoint | `best.pth`，SHA256 `cf26062d9150f227962f76dcb4857533270605bd7f7d7c88d39ef7ef87af3d77` |
| 分歧 Top-K | K = 5 |
| 分歧候选数 | M = 4，BGE/TSE 两个方向各 2 张 |
| 最大交互轮数 | 3 |
| 每轮调用 | 1 次问题生成 + 1 次目标图像代理回答 |
| 最终分数 | `(BGE + TSE) / 2` |
| 提示词版本 | `ddi-qwen-v1` |
| 查询语言 | 英文 |
| 回答策略 | 可确认事实写入查询；不可见或不确定时不更新 |

所有结果均为 `200-query subset`，不能与使用完整 2,000 条测试文本的公开论文结果直接等价比较。

## 9. 定性观察

### 9.1 成功案例

- Query 181：原始目标排名 63。两个模型都围绕包或手持物提问，并将目标提升到排名 18。
- Query 199：`qwen3-vl-flash` 将目标从 64 提升到 45；`qwen3.6-flash` 通过确认“没有手持物”将其提升到 23。

### 9.2 失败与漂移案例

- Query 26（`qwen3-vl-flash`）：原描述已经说明蓝色裤子，新增“裤子为深色”与原查询形成冲突，目标排名从 7 降到 22。
- Query 181 第 2 轮（`qwen3-vl-flash`）：再次确认黑色包属于重复事实，目标从 18 回退到 23。
- Query 198（`qwen3.6-flash`）：新增“棉服/蓬松设计”和“不携带包”后，目标先从 452 降到 488，再降到 625。

这些案例表明，多轮性能下降主要来自三类问题：重复提问、与原查询冲突的视觉判断，以及对低排名困难查询追加不可靠事实。

## 10. 最终结论与建议

### 10.1 研究结论

- BGE–TSE 分歧是有效的交互依据。两个模型的最小消融均显示，DDI 在 Rank-1 和 mAP 上优于通用候选交互。
- MLLM 更强并不意味着多轮结果必然更稳定。`qwen3.6-flash` 的确认率更高，但也更容易把不可靠或重复事实写入查询。
- 交互收益主要集中在第一轮。旧模型第二轮只有小幅追加收益，新模型第二轮已经出现明显回退。
- Overlap@5 应与 Rank-1 和 mAP 联合解释。分支更一致不一定更正确，分支一致性下降也不一定代表检索退化。

### 10.2 推荐配置

当前推荐的最终配置为：

| 项目 | 推荐值 |
|---|---|
| MLLM | `qwen3.6-flash-2026-04-16` |
| 交互轮数 | 1 |
| K / M | 5 / 4 |
| 问题策略 | BGE–TSE 双向分歧候选 + TSE token 线索 |
| 回答策略 | 仅写入明确可见的单条原子事实 |
| 最终推理 | RDE 原生 `(BGE + TSE) / 2` |

若继续优化，优先方向不是增加轮数，而是加入事实冲突检测、重复属性过滤和回答可靠度门控，以减少 mINP 下降和困难查询漂移。
